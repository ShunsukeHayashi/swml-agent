## 他のエージェントと何が違うのか

すべてのコーディングエージェント — Claude Code、Cursor、vibe-local — は基本的に同じ構造だ。

```
while True:
    response = llm(prompt)
    if response.has_tool_call:
        execute(tool_call)
    else:
        break
```

プロンプトを送り、ツールを呼び、また送る。この繰り返し。

SWML-Agentも同じループを持っている。しかし、その上に**物理学のレイヤー**が載っている。

### Ω空間（オメガ空間）

エージェントの状態を4つのフェーズで記述する：

| フェーズ | T（運動エネルギー） | V（ポテンシャル） | H（ハミルトニアン） |
|---------|-------------------|------------------|-------------------|
| OBSERVE | 0.3 | 0.9 | 1.2 |
| PLAN | 0.5 | 0.5 | 1.0 |
| EXECUTE | 0.9 | 0.2 | 1.1 |
| VERIFY | 0.2 | 0.1 | 0.3 |

- **T（運動エネルギー）** = エージェントが実際に動いている度合い。EXECUTEで最大。
- **V（ポテンシャルエネルギー）** = 不確実性。OBSERVEで最大（何もわかっていない）。
- **H = T + V** = ハミルトニアン（全エネルギー）。VERIFYで最小（基底状態）。

### 作用積分

物理学で最も深い原理の一つ、**変分原理**をエージェントに適用する。

```
S = ∫ (T - V) dt
```

自然界では、光は最短経路を選び、惑星は最小作用の軌道を描く。SWML-Agentも同じ原理で動く。タスクの実行経路を最小作用で最適化する。

### 効率メトリクス

```
η = 1 - |S_actual| / |S_worst|
```

最悪ケース（ずっとOBSERVEに留まる）と比較して、どれだけ効率的だったかを百分率で表示する。実際に「Pythonでじゃんけんゲーム作って」と依頼したところ、η=94%が出た。

## なぜ物理学が必要なのか

「ハミルトニアンとか作用積分とか、それって実際に役立つの？単なるメタファーじゃないの？」

正当な疑問だ。現時点では、確かにメタファーの要素が強い。しかし、メタファーには力がある。

### 1. 可観測性

通常のエージェントは「何を考えているか」がブラックボックスだ。SWML-Agentは毎ターン、H、S、ηを表示する。エージェントが迷っているのか（Hが高い）、収束に向かっているのか（Hが下がっている）が一目でわかる。

### 2. 設計原理

「最小作用の原理」は設計判断の指針になる。エージェントに「考えすぎるな」「最短経路を選べ」と自然言語で指示する代わりに、「Sを最小化しろ」と数式で記述できる。曖昧さが減る。

### 3. 将来の最適化

現在のフェーズエネルギーは手動で設定している。しかし、十分なログが溜まれば、実際のタスク実行データからエネルギー関数を学習できる。どのフェーズにどれだけ時間をかけるのが最適かを、データ駆動で決定できるようになる。

## vibe-local との比較

落合さんのvibe-localは素晴らしいプロジェクトだ。教育用途に特化し、7400行で全機能を網羅している。

| | vibe-local | SWML-Agent |
|--|-----------|------------|
| 行数 | 7400 | 1227 |
| 目的 | 教育 | 研究・実験 |
| 理論的基盤 | なし | SWML（ハミルトニアン力学） |
| エネルギー追跡 | なし | H, S, η, エントロピー |
| ダッシュボード | なし | あり |
| サブエージェント | あり | あり |
| セッション保存 | あり | あり |

競合ではなく、異なるアプローチだ。vibe-localは「AIコーディングエージェントとは何か」を教える。SWML-Agentは「AIコーディングエージェントを物理学で記述するとどうなるか」を示す。

## これからの話

SWML-Agentは始まりに過ぎない。次のステップ：

1. **エネルギー関数の学習** — 実行ログから最適なフェーズエネルギーを推定
2. **変分プラン選択の本格実装** — 複数の実行計画をS値で比較し、最小作用の経路を自動選択
3. **ベンチマーク** — SWE-bench等で定量評価。物理学フレームワークが実際に性能向上に寄与するか検証
4. **SWML論文との統合** — Ω関数の完全実装

コードは全て公開している。試したい人はOllamaを入れて `python swml-agent.py` するだけだ。

## 最後に

同じ数学が粒子物理学を記述し、AIエージェントを記述する。

これは偶然ではない。変分原理は「最適な経路を見つける」という問題の普遍的な解法だ。光の屈折も、惑星の軌道も、エージェントのタスク実行も、本質的には同じ問題だ。
